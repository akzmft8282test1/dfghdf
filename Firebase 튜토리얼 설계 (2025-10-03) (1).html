Google 로그인
<pre><code>dartimport &#39;package:google_sign_in/google_sign_in.dart&#39;;

Future&lt;UserCredential&gt; signInWithGoogle() async {
final GoogleSignInAccount? googleUser = await GoogleSignIn().signIn();
final GoogleSignInAuthentication? googleAuth = await googleUser?.authentication;

final credential = GoogleAuthProvider.credential(
accessToken: googleAuth?.accessToken,
idToken: googleAuth?.idToken,
);

return await FirebaseAuth.instance.signInWithCredential(credential);
}</code></pre>

🔹 9단계. Firestore 보안 규칙 설정
개발 중에는 테스트 모드로 Firestore를 열어두었지만, 앱을 서비스할 때는 보안 규칙이 반드시 필요합니다.
NoiseTime은 로그인 사용자만 데이터를 읽고 쓸 수 있도록 제한해야 합니다.

✅ 1. Firestore 보안 규칙 예시
Firebase Console → Firestore Database → 규칙 탭에서 아래 코드 적용:
<pre><code>javascriptrules_version = &#39;2&#39;;
service cloud.firestore {
match /databases/{database}/documents {

// 소음 샘플 데이터
match /noise_samples/{docId} {
allow read, write: if request.auth != null;
}

// 댓글
match /comments/{docId} {
allow read: if true; // 누구나 댓글 읽기 가능
allow write: if request.auth != null; // 로그인 사용자만 작성 가능
}

// 좋아요
match /likes/{docId} {
allow read: if true;
allow write: if request.auth != null;
}

// 그 외 문서는 차단
match /{document=**} {
allow read, write: if false;
}
}
}</code></pre>

✅ 2. 규칙 설명


noise_samples → 로그인된 사용자만 저장/조회 가능


comments, likes → 모두 읽기 가능, 쓰기는 로그인 필요


기타 문서 → 전부 차단



✅ 3. 테스트 방법
Flutter 코드에서 데이터 저장 후, Firebase Console에서 문서를 직접 읽어보세요.
로그인하지 않은 상태에서는 쓰기/읽기 불가해야 정상입니다.
